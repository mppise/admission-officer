# C02 — Student Profile Builder: Implementation Specification

---

## Interfaces

### Public API

```typescript
export function buildStudentProfile(): Promise<{ slug: string; profilePath: string }>;
export function editStudentProfile(slug: string): Promise<void>;
export function showStudentProfile(slug: string): Promise<{ mdPath: string }>;
export function deleteStudentProfile(slug: string): Promise<void>;
export function listStudentProfiles(): Promise<{ slug: string; lastUpdated: string }[]>;
```

---

## File Paths

```
university-ao/students/
  └─ <slug>/
     ├─ profile.json       # Complete structured data (read/write by this component)
     └─ profile.md         # Human-readable export (generated by this component)
```

**Slug format:** Lowercase, alphanumeric + hyphens; generated from student name (e.g., "alice-smith").

## Error Handling

| Error | Recovery |
| :----- | :------- |
| Missing profile.json | Show "Profile not found" + offer to create new |
| Invalid JSON | Show error message + offer to delete and recreate |
| Disk full on write | Show error + suggest delete old profiles |
| Invalid GPA (< 0 or > 5) | Re-prompt with error message |

## Operational Requirements

### UX

- **Field progression:** Step through fields in logical order (academics → tests → activities → awards).
- **Skippable fields:** User can press Enter to skip optional fields.
- **Edit mode:** Re-entering C02 with existing profile pre-populates fields.
- **Confirmation:** Show summary before final save; ask "Save changes?"
- **Response time:** Each field prompt appears < 500ms; saves complete < 2 seconds for local filesystem.

### Data

- **Atomicity:** Write new profile to temp file, then atomic rename; never partial files.
- **Encoding:** All text UTF-8.
- **Timestamps:** ISO 8601 format (e.g., 2026-05-31T12:00:00Z).
- **Slug stability:** Slug is generated once and stable (used in file paths); never change slug without migration.

### Security

- **No PII exposure:** Profiles stored locally; no upload to cloud.
- **File permissions:** Profile files readable/writable by owner only (mode 600).

## Testing Requirements

**Coverage:** 80% line coverage.

**Critical paths:**
- [ ] New profile creation: Empty → all fields → save → JSON + Markdown valid
- [ ] Edit existing profile: Load JSON → modify field → save → round-trip preserves data
- [ ] Slug generation: Student name with spaces/punctuation → valid slug
- [ ] Field validation: Invalid GPA → error → re-prompt
- [ ] Deletion: Remove folder + all files
- [ ] Large transcript: 8 years × 10 courses → no crashes, JSON valid
