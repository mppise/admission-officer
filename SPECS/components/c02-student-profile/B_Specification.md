## 2. Error Handling

| Scenario | Error Message | Recovery |
|:---|:---|:---|
| Name is empty | "Student name cannot be empty." | Prompt user to enter name |
| Invalid GPA | "GPA must be a number between 0 and 4.5." | Prompt user to correct |
| File write fails | "Could not save profile: [error]" | Suggest checking disk/permissions; allow retry |
| Gemini markdown generation fails | "Could not generate markdown summary. Continuing without it." | Profile.json saved; user can retry markdown export later |
| Profile directory not found (edit) | "[slug] not found." | Return to student list |
| Delete confirmation timeout | (user exits mid-confirmation) | No deletion; return to menu |

## 3. Operational Requirements

### 3.1 UX Patterns

- **Form flow:** Sequential field prompts via waitForText() (name, grad year, GPA, etc.)
- **Multi-item fields:** Add extracurricular / award / etc. via "Add another?" confirmation loop
- **Optional fields:** Skip with empty input (tracked in fieldStatus)
- **Progress indication:** Spinner while saving JSON or generating markdown

### 3.2 Data Validation

- **Name:** Non-empty, max 100 chars
- **Grad year:** Valid year (2020–2030)
- **GPA:** Float 0.0–4.5
- **Test scores:** Valid ranges (SAT 400–1600, ACT 1–36)
- **Major names:** Non-empty strings, max 100 chars
- **Slug generation:** toSlug(name) → lowercase, spaces → hyphens

### 3.3 File I/O

- **Save:** workspace/students/{slug}/profile.json (formatted JSON, UTF-8)
- **Export:** workspace/students/{slug}/profile.md (markdown via Gemini)
- **Directory structure:** Auto-created via ensureDir()
- **File permissions:** Standard (0755 dirs, 0644 files)

### 3.4 Security

- **Input sanitization:** Reject XSS-like strings? (not critical for local CLI, but good practice)
- **No PII exposure:** Don't log full names or test scores to stdout
- **File permissions:** Workspace should be readable by owner only (user must chmod if needed)

### 3.5 Scalability

- **Profile size:** ~50KB per JSON + ~20KB per markdown
- **No limits:** Can store unlimited students (bounded by disk)

## 4. Testing Requirements

**Coverage threshold:** ≥80% line coverage

### Critical Paths

1. **Create profile:** Form intake → validation → JSON save → markdown export
2. **Edit profile:** Load JSON → update field → re-save → verify JSON updated
3. **Delete profile:** Confirmation → directory deletion → verification
4. **Error handling:** Missing file → clear error → user can retry
