## Key Facts for Developers
- C05 collects the essay prompt **interactively via tui.tsx** (`SpaciousSelect` + `waitForText`) — not from a CLI flag. C01 dispatches with `studentSlug`, `uniSlug`, and a `timestamp`.
- `enquirer` is **NOT used** in C05 — all prompts use `tui.tsx` helpers exclusively.
- Temperature is `0.8` — highest in the system; this is a creative writing task.
- Sample count is **dynamic**: 2 for prompts < 500 words, 3 for ≥ 500 — enforced by the Gemini prompt instruction.
- The **⚠️ disclaimer** must appear in BOTH the saved markdown AND the CLI stdout after save — two separate placements.
- The overwrite-check slug combines `essayTypeSlug` + first 6 chars of a **djb2 hash** of the prompt text. The **directory name** is always the `YYYY-MM-DD-HHmm` timestamp from C01.
- The `timestamp` is **generated by C01** and passed into `buildEssay` — C05 does not generate it.
- `showEssay` receives the `timestamp` from C01 (C01 calls `listEssays` and presents a selection menu).
- All file paths use **`C07.workspacePath()`** — essay data lives at `university-ao/students/<s>/universities/<u>/essays/<timestamp>/`.
- Student profile + essay prompt content is sent to Google's Gemini API — same posture as C04.
