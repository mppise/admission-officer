# C01 CLI Shell — Spec Reading Guide

## Reading Order
1. `../../artifacts/B_Architecture.md` — system constraints (mandatory first)
2. `A_Core_Spec.md` — what to build and why (includes full menu flow diagram)
3. `B_Interfaces.md` — exact function contracts to implement
4. `C_Operational_Specs.md` — screen layouts, UX detail, error handling

## Authority Rules
- If A_Core_Spec and B_Interfaces conflict: B_Interfaces wins for signatures; A_Core_Spec wins for behavior.
- If any spec conflicts with B_Architecture.md: stop and raise with DevLead before proceeding.
- Do not infer missing details — raise as a spec gap.

## Key Facts for Developers
- C01 source is `src/components/c01-cli-shell/index.tsx` — NOT `src/cli/index.ts` (old path, deleted).
- C01 calls `await bootstrap()` from C07 **before** any ink render.
- C01 **never** reads or writes data files directly — all I/O is delegated to C02–C07.
- All user-supplied names and domains are sanitised to slugs by C01 before passing to components.
- Timestamps (`YYYY-MM-DD-HHmm`) are generated by C01 and passed to C04/C05.
- The API key check (before dispatching to AI-dependent components) is C01's responsibility — if missing, direct user to Config screen.
- No `commander`, no flags, no `process.argv` parsing beyond `process.argv[2]` check (not needed).
