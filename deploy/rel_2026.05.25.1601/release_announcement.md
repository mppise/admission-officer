# Release Announcement — university-admission-officer v1.1.0

**Date:** 2026-05-25
**Type:** Minor release
**Audience:** Users / Operators

---

## What's New

### Student Profile — Complete Redesign (C02)

The student profile experience has been rebuilt from the ground up.

**Before:** A linear wizard that asked every question in sequence. If you made a mistake, you had to restart. If the app was interrupted mid-way, all unsaved data was lost. Editing an existing profile would silently overwrite complex fields (transcript, test scores, activities, awards) with empty values.

**Now:**
- **Nested menu navigation** — browse to any section or field at any time. Edit one field, leave everything else untouched. Go back freely.
- **Per-field completion tracking** — every field shows its status (`○ not started`, `● pending`, `✓ complete`, `– skipped`) so you always know what's left.
- **Finalization gate** — the profile is only written when you explicitly choose "Finalize & Save" and every field is either set or skipped. No accidental partial saves.
- **Incremental auto-save** — `profile.json` is written after every individual input. Closing the app mid-session loses nothing.
- **AI-enhanced output** — at finalization, Gemini reviews your raw inputs, fixes spelling and grammar, and reframes your activity and award descriptions as concrete strengths in your own voice. No marketing language, no invented facts. Your raw data is always preserved in `profile.json`.

### List management for activities, awards, transcript, and test scores

All variable-length fields (extracurriculars, awards, transcript years/courses, AP/IB scores) now support Add / Edit / Remove per individual entry from a sub-menu.

### ESLint added

`npm run lint` now available. TypeScript linting with `@typescript-eslint/recommended` rules.

## Required Actions

**Existing profiles (`profile.json` from a previous session):**
No migration required. v1.1.0 reads and resumes existing `profile.json` files correctly. If your `profile.json` contains a `personalStatementSummary` field (from a pre-release session), it will be ignored — it does not affect functionality.

**Operators / fresh install:**
No new environment variables required beyond what v1.0.x already needed (`GEMINI_API_KEY`, `GEMINI_MODEL`, `GEMINI_TOKEN_WINDOW`). The Gemini enhancement call uses the same credentials.

---

## Known Limitations

- No undo within a session — once a field is saved to `profile.json`, it persists. To revert, re-open the field and re-enter the previous value.
- Gemini enhancement is a best-effort step. If the API is unavailable at finalization, `profile.md` is generated from your raw input with a warning message. No data is lost.
- Existing `profile.md` files generated by v1.0.x are not retroactively enhanced. Re-run `ao --student-profile --build --name <slug>` and select Finalize & Save to regenerate with enhancement.
